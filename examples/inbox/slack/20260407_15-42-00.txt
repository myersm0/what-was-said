dev-team  [4:42 PM]

Alice Wilson  [4:49 PM]
Found it. In `incremental_build.py` line 253, the memoization key is built from `target.name` and `target.config`, but it's not including the `target.scope`. This means if you have the same target name in different scopes, they'll get the same cache entry, and subsequent lookups will return the cached result from a different scope entirely. That's a classic cache key collision bug.
